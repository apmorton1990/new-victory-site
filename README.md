# New Victory — Website

The website for New Victory church, built with **Statamic** (free Core edition)
as a flat-file CMS with a visual page builder, and published as a **static
site** that can be hosted anywhere — including one.com shared hosting.

- Authoring/build tool: **Statamic 5 (Core, free)** — Laravel/PHP, runs locally
  (or in CI), not on the production host.
- Production: **static HTML/CSS/JS** generated by Statamic's static site
  generator and uploaded to the host. No PHP runtime required in production.

```
new-victory-site/
└── cms/                              # Statamic (Laravel) app — author + build here
    ├── content/                       # flat-file content (committed)
    │   ├── collections/pages/*.md      # pages + their page_builder blocks
    │   ├── globals/default/settings.yaml  # site settings
    │   └── navigation/ + trees/        # menus
    ├── resources/
    │   ├── blueprints/                 # field definitions (page builder, settings)
    │   ├── views/                      # Antlers templates
    │   │   ├── layout.antlers.html      # header/footer (settings + nav)
    │   │   ├── page.antlers.html         # renders the page builder
    │   │   └── blocks/                    # one partial per block: hero/text/image/cta
    │   └── css/site.css                # Tailwind v4 + brand tokens
    └── storage/app/static/            # GENERATED static site (git-ignored)
```

## Prerequisites (for editing / building — not the production host)

- **PHP 8.3+ and Composer** via **[Laravel Herd](https://herd.laravel.com)**
  (Herd's binaries live in `~/.config/herd/bin`).
- **Node.js 20+** (for the Tailwind/Vite asset build).

## Local setup

```bash
cd cms
composer install
npm install
cp .env.example .env          # then: php artisan key:generate
php please make:user          # create a control-panel login
```

## Editing content

```bash
cd cms
npm run dev                   # Vite (Tailwind) dev server, for styling
php artisan serve             # Statamic at http://127.0.0.1:8000
```

- **Site preview:** http://127.0.0.1:8000
- **Control panel:** http://127.0.0.1:8000/cp — where the team builds pages

### Collections

- **pages** — marketing pages, built with the page builder. Home renders at `/`.
- **ministries** — VKids, Relentless Youth, etc. Reuse the same page builder and
  route at `/ministries/{slug}`, plus meta fields (summary, image, age range, leader).
- **staff** — team members (name, role, photo, bio); surfaced by the `team_grid` block.

### Page builder

The `page_builder` field is a shared **fieldset** (`cms/resources/fieldsets/page_builder.yaml`)
imported by both `pages` and `ministries`. Each block "set" maps to an Antlers
partial in `cms/resources/views/blocks/`:

- **Layout/content:** hero, text, image, cta, feature_cards, image_text,
  accordion, feature_list, steps, tiers, button_group
- **Media:** video (YouTube/Vimeo), gallery, embed (raw HTML — Church Center / Mailchimp / maps)
- **Church-specific:** leaders_feature, team_grid (pulls the `staff` collection),
  service_times + location_map (pull the Service Info global), quote

### Globals & navigation

- **Settings** — site title, contact, socials, footer text.
- **Service Info** — service times, schedule note, address, map link (used by the
  `service_times` and `location_map` blocks).
- Menus come from the **Main** and **Footer** navigations.

> **Adding a block type:** add a set to `resources/fieldsets/page_builder.yaml`
> and a matching partial `resources/views/blocks/{type}.antlers.html`.

## Building & deploying the static site

```bash
cd cms
npm run build                 # compile Tailwind/JS into public/build
php please ssg:generate       # render static site -> cms/storage/app/static/
```

Then **upload the contents of `cms/storage/app/static/`** to your host's web
root (e.g. one.com via SFTP / file manager). It's plain static files — no PHP,
no database, no SSH needed on the host.

> **Production base URL:** set `APP_URL` in `cms/.env` (or the SSG `base_url`
> config) to the live domain before generating, so links/assets resolve.

## Planning Center

Because production is static, dynamic Planning Center data is handled one of two
ways (decided per feature):

- **Embeds** — drop a Church Center widget (calendar, signups, giving) into a
  page/block; it loads client-side, no rebuild needed.
- **Scheduled rebuild** — a build process (locally or in CI) pulls PCO via its
  API at build time and re-generates + re-uploads the static site on a schedule.
  Statamic Core is full Laravel, so a custom Antlers tag + cached HTTP call is
  the clean pattern when we add it.
